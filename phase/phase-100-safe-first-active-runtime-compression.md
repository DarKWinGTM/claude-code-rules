## Objective

Reduce repeated explanation, recap, examples, and secondary reminder blocks across selected active runtime rules while preserving all execution-relevant mechanisms and keeping the active runtime install set at 18 root rules.

---

## Why This Phase Exists

The active merged RULES set is already functionally strong, but recent audit work shows a modest safe-first compression opportunity across repeated examples, repeated anti-pattern summaries, repeated checklists, and repeated reminder blocks.

P100 exists to capture those savings without broad deletion and without weakening owner boundaries such as worker-routing versus safe-io, phase-backed task semantics, refusal/recovery schema, portability drift response, or evidence-strength wording behavior.
