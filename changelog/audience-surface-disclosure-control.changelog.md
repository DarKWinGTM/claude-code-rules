## Version 1.0: Created direct-user-transparent audience-surface disclosure owner

**Date:** 2026-05-07
**Session:** d42465eb-30a7-4bc8-b9d6-03e52306e9a5

### Changes
- Created `audience-surface-disclosure-control.md` as the active runtime owner for audience-surface disclosure behavior.
- Created `design/audience-surface-disclosure-control.design.md` as the active target-state design companion.
- Defined direct authorized user/project-owner communication as full, transparent, evidence-calibrated disclosure.
- Defined audience classification for internal engineering/governance, operator-facing, customer/public, logs/demos, and externally shared surfaces.
- Added minimization guidance for secrets, credentials, private endpoints, supplier/provider identities, raw user data, internal security mechanisms, and unnecessary implementation details in generated audience-facing artifacts.
- Clarified that audience minimization must preserve useful public clarity and must not hide material blockers, risks, or verification limits from the direct user.

### Summary
Audience-surface disclosure control now separates full direct-user transparency from safer generated public/operator/customer/log/demo/external copy so internal details are not placed on the wrong audience surface.
