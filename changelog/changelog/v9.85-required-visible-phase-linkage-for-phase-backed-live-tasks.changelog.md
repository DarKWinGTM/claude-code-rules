## Version 9.85: Required visible phase linkage for phase-backed live tasks

**Date:** 2026-05-04
**Session:** d42465eb-30a7-4bc8-b9d6-03e52306e9a5

### Changes
- Completed the P076-03 phase-visible task linkage refinement under the existing P076 design-to-phase execution synthesis family.
- Updated `phase-implementation.md`, `design/phase-implementation.design.md`, and `changelog/phase-implementation.changelog.md` to v2.27 so non-trivial phase-backed built-in task entries must visibly link to active or clearly implied phase context.
- Updated `todo-standards` to v2.22 and `artifact-initiation-control` to v1.8 so phase-shaped task creation, extension, and startup initialization preserve visible phase linkage in the subject or description.
- Updated `project-documentation-standards` to v2.33 and `execution-continuity-and-mode-selection` to v1.11 so built-in task lists remain live tracking rather than phase authority while continuation-created or continuation-extended tasks do not become generic next-work entries.
- Created `phase/phase-076-03-phase-visible-task-linkage.md` and `patch/phase-visible-task-linkage.patch.md` to track the governed refinement and release gates.
- Preserved the active runtime install count at 43 because P076-03 refines existing owners rather than adding a new runtime rule.

### Summary
P076-03 closes the practical gap where phase-backed work could be planned correctly but still appear as generic built-in task-list entries during execution. The release keeps phase semantics under `/phase`, keeps task lists as live execution pointers, and requires visible phase linkage through subject or description so active phase context survives startup and continuation.
