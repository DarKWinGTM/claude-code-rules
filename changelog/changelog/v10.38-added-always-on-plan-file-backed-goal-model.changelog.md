## Version 10.38: Released always-on plan-file-backed goal model

**Date:** 2026-06-02
**Session:** 1f1873d2-0feb-485f-a5ff-d383254590dd

### Changes
- Opened and closed P130 on top of the released `v10.37 / P129` baseline.
- Refined RULES so any actual governed `/goal` creation now prepares a full detailed plan file before final goal emission.
- Required emitted governed `/goal` output to reference that prepared route file through `Plan reference` or equivalent route-only wording.
- Preserved `/goal` as the objective owner for outcome, proof/checks, scope, and hard guardrails.
- Preserved the referenced plan file as subordinate route support only rather than objective authority or completion proof.
- Preserved `/plan` as explicit standalone planning, later route revision, or overflow only rather than the ordinary paired next surface.
- Synced touched runtime/design/changelog/TODO/phase/patch surfaces for this release.
- Runtime install/update verification, 18/18 source/runtime parity, source/destination body sufficiency, `git diff --check`, push/update to `master`, and GitHub release verification passed.
- Release URL: https://github.com/DarKWinGTM/claude-code-rules/releases/tag/v10.38

### Summary
P130 hardens governed `/goal` authoring into a plan-first model: prepare the detailed route file first, emit a compact objective-owned `/goal` second, and keep the referenced plan file subordinate to the goal rather than a second public objective surface.
