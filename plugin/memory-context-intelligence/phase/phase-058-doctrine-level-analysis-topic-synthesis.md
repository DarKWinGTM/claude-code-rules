## Objective

Lift `/memory-context-intelligence:analysis` topic synthesis through doctrine/mechanism lenses so top-level topics stay at RULES/workflow principle level while incident details remain inside proposal evidence/examples.

## Why this phase exists

Earlier waves already restored historical-first scope, minimum-three advisory behavior, and the permission-safe slash wrapper, but real local output still let some fallback topics read too much like raw recurring-pattern wording or narrow keyword suffixes. This phase exists to keep surfaced topics doctrine-level without weakening the trace-anchored evidence model or dropping sparse-history advisory coverage.

## Gate

Phase 058 closes only when all of the following are true in checked scope:
- focused RED/GREEN tests prove config-investigation traces are lifted into doctrine-level topic titles
- focused RED/GREEN tests prove recurring-pattern fallback no longer surfaces raw `private` / `task` suffix titles
- sparse historical behavior still surfaces at least 3 advisory topics
- proposal output keeps incident details in evidence/examples instead of top-level titles
- the full `memory-context-intelligence` suite passes
- one real local `/memory-context-intelligence:analysis` run shows doctrine-level advisory topics with provenance/confidence boundaries
- governed/runtime-facing docs are synchronized to the doctrine-level contract

## Verification / closeout

Phase 058 is completed in checked scope.

This closeout now holds:
- `lib/signals.py` now lifts config-investigation and recurring-pattern traces through doctrine/mechanism-specific title heuristics before top-level topic naming
- doctrine-level titles now include outputs such as `Verify semantic fit before reusing an existing control surface`, `Separate symptom evidence from strategic fix scope`, `Require source-backed behavior verification before choosing config-only remediation`, `Keep audience disclosure boundaries explicit in surfaced proposals`, and `Turn recurring workflow findings into decision-ready proposals`
- proposal evidence/examples now retain incident details such as `404`, `passStatusCodes`, and `opera_fallback` instead of promoting those tokens into the title
- focused RED/GREEN proof passed for `tests/test_signals.py` with `18` tests green after adding doctrine-level title checks
- focused presentation proof passed for `tests/test_presentation.py` with `12` tests green
- the full runtime/source suite passed with `69` tests green
- a checked local `/memory-context-intelligence:analysis` run with `--permission-mode bypassPermissions` returned three doctrine-level advisory topics with `historical-only` / `low` confidence boundaries visible in the output
- the local package version was updated from `0.9.16` to `0.9.17`

## Boundaries preserved after closeout

Phase 058 still does not claim:
- context-only promotion without trace evidence
- removal of minimum-three advisory behavior for sparse history
- a change to `/additional/`

## Rollback notes

Phase 058 is a topic-synthesis/output-contract refinement layered on top of the earlier historical-default, breadth-ordering, and permission-safe slash-proof waves. Rolling it back would restore narrower/rawer fallback wording while leaving the phase 053, 056, and 057 foundations intact unless a broader rollback is explicitly selected.
