# Version 0.1.3 - Clarified the topic list as a proposal surface for RULES-adjacent improvement directions

> **Parent Document:** [changelog.md](changelog.md)
> **Current Version:** 0.1.10
> **Session:** 20625882-e064-4b40-a415-8d425cf392dc (2026-05-16)

---

## Changes
- updated the main concept design so the consumer list is explicitly described as proposal/analysis lanes rather than immediate RULES edits
- clarified that the first topic list presents candidate improvement topics and candidate mechanisms, not already-selected rule changes
- added a boundary that candidate topics must not be presented as if they were approved RULES mutations
- strengthened the README and topic-flow wording so the user-facing list is easier to understand as a proposal surface

## Summary
This version reduces ambiguity in the concept by making it explicit that the list shown by the plugin is a proposal layer for possible RULES-adjacent improvement directions. The list helps the user decide what to explore next; it does not itself change RULES.
