# Version 0.1.5 - Added research enrichment as a topic-strengthening stage

> **Parent Document:** [changelog.md](changelog.md)
> **Current Version:** 0.1.10
> **Session:** 20625882-e064-4b40-a415-8d425cf392dc (2026-05-16)

---

## Changes
- added `design/research-enrichment.design.md` to define when web research should strengthen locally derived candidate topics
- added research modes: `local-only`, `hybrid`, `research-first`, and `verification-heavy`
- updated topic-list flow so each listed topic can indicate whether external research is likely needed
- added a completed local phase for research enrichment and updated the capsule roadmap
- clarified that research enrichment strengthens proposal quality but does not override user instruction, checked evidence, or active RULES authority

## Summary
This version makes the concept more practical by defining how local work traces and external research can work together. Local traces identify promising improvement topics; focused research then strengthens only the topics that need broader fact, theory, standards, or best-practice support.
