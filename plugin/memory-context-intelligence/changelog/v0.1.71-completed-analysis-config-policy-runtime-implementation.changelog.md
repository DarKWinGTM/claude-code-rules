## Summary

Completed phase 067 as the bounded runtime config-policy implementation wave for `memory-context-intelligence`: the analysis path now loads late-bound source policy from explicit `--config` or upward-discovered project config files, applies source-selection/source-limit precedence before `intake → signals → present`, surfaces policy-limited provenance honestly, and emits an advisory guided config helper when no config file is loaded.

## Changes

- added `lib/config_policy.py` as the source-policy loader and guided config-question helper owner
- updated `lib/intake.py` so source policy can limit source classes, cap historical shards, and request same-day widening only for non-explicit runs
- updated `lib/signals.py` and `lib/presentation.py` so policy-limited runs keep their limitation visible and cannot silently overread weaker evidence as broader proof
- updated `lib/analysis_surface.py` so `/memory-context-intelligence:analysis` forwards explicit config paths, exposes `source_policy`, and surfaces `config_questions` plus `suggested_config_path` when no config file is loaded
- updated `bin/memory-context-intelligence`, `skills/analysis/SKILL.md`, design, phase, changelog, patch, and root TODO surfaces to the implemented bounded behavior

## Boundary

This version keeps config policy bounded. It does not claim automatic config writing, selected-topic persistence in config, a fifth evidence class, semantic-authority promotion for config, `/additional/` changes, plugin-managed auto-flow proof, publication, external marketplace release, stable/broad readiness, or main RULES promotion/mutation/merge.
