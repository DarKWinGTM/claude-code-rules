## Changes

- completed phase 009 by adding internal signal extraction and topic generation to the isolated runtime package under `<repo-root>/plugin/memory-context-intelligence/`
- extended the package command surface with a `signals` subcommand that consumes bounded intake JSON from stdin or `--intake-report <path>`
- added `lib/signals.py` as a no-dependency helper for clustering duplicate/near-duplicate intake records, ranking signals, grouping recurring patterns/corrections/blockers/evidence gaps, and emitting internal topic candidates
- added topic candidate fields for purpose, why surfaced, expected behavior impact, high-level mechanism, expected output, confidence, evidence label, and source signal IDs
- added confidence/evidence handling that keeps stale, insufficient, weak, and single-observation input low-confidence or unpromoted
- added focused tests under `tests/test_signals.py` for controlled sample output, duplicate merge behavior, stale/insufficient handling, and no-emission flags
- synced package README, design, changelog, phase, patch, skill, trace-scout, and plugin metadata to the phase-009 internal-analysis boundary
- updated capsule README, design parent, phase 009, phase summary, patch, and root TODO wording to show phase 009 complete and phases 010+ still pending

This version closes phase 009 by making the first analysis layer real but internal-only.
